# Summary: Applied Cryptography (Bruce Schneier)
## Sections 2.3 & 2.4: One-Way and Hash Functions


---

### 2.3 One-Way Functions
A **One-Way Function** is the mathematical equivalent of a "trapdoor." It is designed to be easy to enter but nearly impossible to escape without specific information.

* **The Directional Gap:** * `Input → Output`: Computationally fast and easy.
    * `Output → Input`: Computationally infeasible (takes too much time/power).
* **Security Foundation:** Security isn't based on "mathematical impossibility" but on **computational difficulty**. If it takes a supercomputer 1,000 years to reverse a function, it is effectively secure for today.
* **Core Applications:**
    * Password protection.
    * Digital signatures.
    * Public-key cryptography (e.g., factoring large primes).



---

### 2.4 One-Way Hash Functions
A **Hash Function** takes a variable-length input (a word, a book, or a movie) and compresses it into a **fixed-length** string of bits.

#### Key Requirements
1.  **Efficiency:** It must be easy and fast to compute.
2.  **Preimage Resistance:** Given a hash, it should be impossible to find the original input.
3.  **Collision Resistance:** It should be nearly impossible to find *any* two different inputs that produce the same hash.
4.  **The Avalanche Effect:** A tiny change in the input (like changing a comma to a period) results in a radically different hash.



#### Security Properties
| Property | Description |
| :--- | :--- |
| **Preimage Resistance** | Hard to find the original input from the hash. |
| **Second Preimage Resistance** | Hard to find a *different* input that results in the same hash as a specific input. |
| **Collision Resistance** | Hard to find *any* two inputs that match. |

#### Why it Matters
The main takeaway is **Data Integrity**. By storing the *hash* of a password rather than the password itself, systems ensure that even if a database is compromised, the actual credentials remain obscured.
